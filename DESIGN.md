# DESIGN.md

이 파일은 디자인 및 퍼블리싱 규칙을 정의합니다. 디자인 컨펌/피드백이 있을 때마다 업데이트합니다.

## 프로젝트 개요

- **Ai-Design** / 디자인 토큰 기반 UI 퍼블리싱 + 개발 프로젝트
- 타겟 플랫폼: Web (반응형 — PC 우선, 모바일 대응)
- 프레임워크: React + TypeScript + Tailwind CSS v4 + shadcn/ui
- 디자인 토큰: CSS Variables 기반 (Style Dictionary로 생성)

## 디자인 원칙

1. 15년차 시니어 프로덕트 디자이너 수준의 레이아웃과 시각적 완성도를 유지한다.
2. 공백(spacing)과 타이포그래피 위계(hierarchy)가 명확해야 한다.
3. 컴포넌트는 반드시 디자인 토큰(CSS Variables)을 사용하고, 하드코딩된 색상/수치는 금지한다.
4. 기능 구현은 하지 않는다. UI 퍼블리싱(정적 마크업 + 스타일)만 진행한다.
5. shadcn/ui 컴포넌트를 최대한 활용하되, 필요 시 커스터마이징한다.
6. 모든 컴포넌트는 Light / Dark 모드를 지원해야 한다.
7. 한 번의 프롬프트에 한 페이지(또는 한 섹션)씩만 작업한다.

## 레이아웃 구성

- **RootLayout**: 전체 앱 래퍼 (data-theme 속성 관리, 폰트 설정)
- **Header**: 로고 + 네비게이션 + 유저 영역 (반응형)
- **Sidebar**: 사이드 네비게이션 (필요 시 추가)
- **PageWrapper**: 페이지 콘텐츠 최대 너비 + 기본 패딩 래퍼

## 퍼블리싱 규칙

### 색상/간격

- 모든 색상/간격은 CSS Variable 또는 Tailwind 토큰을 사용한다.
- 하드코딩된 색상값(`#`, `rgb` 등)은 절대 사용하지 않는다.
- Light/Dark 전환은 `.dark` 클래스로 제어한다.
- 다크모드가 기본이다. 페이지 루트에 `.dark` 클래스를 적용한다.
- 브랜드 컬러(`--primary`)는 CTA 버튼뿐 아니라 로고/심볼, 회원가입 같은 핵심 링크, 포인트 요소에도 사용하여 페이지 전체에 아이덴티티가 드러나도록 한다.

### 컴포넌트

- 컴포넌트마다 JSDoc 주석으로 역할을 명시한다.
- 같은 역할의 UI가 다른 코드로 중복 작성되지 않도록 한다.
- 로그인 폼 등 단일 목적 레이아웃은 Card로 감싸지 않는다. Card는 대시보드/리스트 등 콘텐츠 그룹핑이 필요한 경우에만 사용한다.

### 버튼

- 같은 그룹의 버튼은 동일한 정렬(align) 방식을 사용한다. 한 그룹 내에서 중앙/좌측 정렬이 섞이지 않도록 한다.
- SNS 로그인 버튼은 `variant="outline"`을 사용하고, 아이콘 + 텍스트를 중앙 정렬한다.

### 반응형 대응

1. 그리드: Desktop 3열 → Tablet 2열 → Mobile 1열
2. 사이드바: Desktop 고정 → Mobile 햄버거 메뉴로 전환
3. 폰트 사이즈: 타이틀/헤딩은 모바일에서 한 단계 축소
4. 패딩/마진: 모바일에서 horizontal padding 줄이기
5. 테이블 → 모바일에서 카드형 리스트로 전환

## 디자인 일관성 검수 체크리스트

1. **색상** — 하드코딩된 색상값이 있는지 (있으면 토큰으로 교체)
2. **간격** — spacing 토큰 외의 임의 수치가 있는지
3. **타이포그래피** — font-size/weight/line-height가 토큰 기반인지
4. **컴포넌트** — 같은 역할의 UI가 다른 코드로 중복 작성된 부분이 있는지
5. **접근성** — aria-label, alt 텍스트 누락 여부
6. **Dark mode** — `.dark` 클래스 또는 CSS Variable이 올바르게 적용됐는지
7. **정렬 일관성** — 같은 그룹의 버튼/요소가 동일한 정렬 방식을 사용하는지

## 금지 패턴

> 디자인 피드백으로 확인된 안티패턴. 절대 반복하지 않는다.

- 단일 목적 페이지(로그인 등)를 Card로 감싸지 않는다.
- 같은 그룹의 버튼에 서로 다른 정렬(중앙/좌측)을 섞지 않는다.
- 브랜드 컬러를 CTA 버튼 하나에만 넣고 나머지는 무채색으로 방치하지 않는다. 로고/심볼, 핵심 링크 등에 브랜드 컬러를 분산 배치한다.
- 다크모드 우선 앱에서 라이트 배경을 기본으로 보여주지 않는다.
