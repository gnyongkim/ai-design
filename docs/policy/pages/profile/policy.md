## 🎯 1. 핵심 정책 규정 (UX Core)

### 1-1. '자랑하고 싶게 만드는' 통계 (Bragging Rights)
- 단순 텍스트 나열식의 통계를 지양하고, 게이미피케이션(배지, 티어, 트로피 마이리지) 요소를 적극 이용해 이미지로 쉽게 캡처/공유할 수 있는 모듈을 제공합니다.

### 1-2. 장비 관리 특화 (Gear Tracking)
- 일반적인 스포츠 앱을 넘어 러너/사이클리스트들의 핵심 니즈인 '장비 수명' 관리(예: 현재 신고 있는 알파플라이3 쿠션 교체 시기)를 개인 맞춤형 대시보드 구조에 포함시킵니다. 커머스 탭과의 리텐션 시너지를 노립니다.

## 📊 2. 데이터 노출 우선순위 (UI 레이아웃 기준)

1. **상단 프로필 (Sticky Header)**: 
   - 거대한 프로필 이미지(또는 아바타)와 현재 '운동 레벨(Tier) 배지', 아이디.
2. **캘린더 & 히트맵 (Heatmap / Calendar) 시각화**:
   - Github Contribution Graph 처럼 올 한해 내가 운동한 빈도를 직관적인 컬러 셀로 나타냅니다. (빈 맵을 채우고 싶은 심리 유발).
3. **히스토리 (History List)**:
   - 각 날짜별 개별 운동 기록들 (지도 썸네일, 뛴 시간, 소모 칼로리).
4. **기어 룸 (Gear Room)**:
   - 내 런닝화 리스트와 각 신발별 쌓인 마일리지 잔여 게이지바.

## 🎨 3. 레이아웃 및 렌더링 가이드라인

### 3-1. 레이아웃 구조
- **모바일**: 탭스뷰(Tabs). 정보가 많으므로 크게 `[나의 기록] | [마이 기어] | [셀팅]`으로 슬라이더 탭을 나누어 세로 스크롤 길이를 통제합니다.

### 3-2. 컴포넌트 시각적 퀄리티 (Senior Polish)
- **히트맵 & 배지**: 디자인 토큰의 `Primary 컬러`를 사용하여, 빈도는 낮을 때는 투명하게(`bg-primary/20`), 높을 때는 강하게(`bg-primary`) 표현되는 토큰 스택을 활용합니다.
- 복잡한 캘린더나 커스텀 차트 적용을 최소화하고, SVG 도표 베이스의 고품질 데이터 비주얼라이제이션(Data Vis)을 구축합니다.
