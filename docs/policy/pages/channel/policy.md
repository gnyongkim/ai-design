# 🏃‍♂️ 채널 (그룹 운동) UI/UX 기획 및 정책서

**설명**: 실시간 액티비티 기반의 유저 간 그룹 운동 매칭(번개/Flash Mob 방식)을 지원하는 소셜 허브 영역에 대한 디자인 및 비즈니스 정책 문서입니다.

---

## 🎯 1. 핵심 정책 규정 (UX Core)

### 1-1. 휘발성 '방폭형' (1회성 오픈룸 기조)
- 채널은 지속적으로 관리해야 하는 '동호회/크루' 개념을 탈피합니다.
- **"오늘 당장 이 페이스로 같이 뛸 사람"**을 구하는 목적에 부합하는 1회성 룸(Room) 구조를 취합니다.
- 운동 일정 시간이 종료되거나 호스트가 채널을 종료하면 폭파(실시간 지도 추적 종료)되는 가벼운 구조입니다.

### 1-2. 트렌딩 우선 편성과 탐색 (Discovery First)
- 사용자가 하단 GNB를 통해 채널 탭 진입 시, **새롭게 가입할 수 있는 트렌딩 채널 리스트(탐색)**가 디폴트로 활성화됩니다.
- "내가 속한 채널"은 버튼이나 탭 분리로 서브 진입구를 마련하여, 앱 내비게이션의 초기 매력을 즉각적인 '매칭 탐색'에 베팅합니다.

---

## 📊 2. 데이터 노출 우선순위 (UI 레이아웃 기준)

디자이너/퍼블리셔가 채널 목록 카드 컴포넌트를 설계할 때 고려해야 할 요소별 시각적 우선순위입니다.

1. **지역 (Location)**: 
   - 실시간 오프라인 운동 성격장 가장 중요한 기준. 카드 상단 혹은 가장 강력한 아이콘(`MapPin`)과 함께 노출 (예: 한강공원, 남산 둘레길).
2. **현재 접속(활성) 인원 (Live Members)**: 
   - 혼자 뛰는 것이 아님을 알리는 강력한 소셜 자극 장치. (예: 🔴 12/20명)
3. **타겟 정보 (Target Pace / Distance)**: 
   - 나와 운동 레벨(속행)이 맞는지 검토하기 위한 필수 수치 (예: 5'30", 10km)
4. **상태 / 태그 (Status Tag)**: 
   - [오픈/잠금], [러닝/사이클] 구분을 컴팩트한 태그 형태로 표시.

---

## 🎨 3. 레이아웃 및 렌더링 가이드라인 (Design Standard)

### 3-1. 레이아웃 구조 (Mobile / Desktop)
- **상위 컨트롤러**: 상단에 [내 채널] / [탐색(트렌딩)] 탭 스위치를 배치하고, 하단에 즉시 필터 가능한 캐러셀 필터 칩(지역, 거리, 페이스) 배치.
- **메인 리스트**: 카드가 1열(모바일) 또는 2-3열(데스크탑 Grid)로 떨어지는 리스트 뷰 구조. 모바일의 경우 `DESIGN.md` 규칙에 따른 `px-5` 컨테이너.
- **방 생성 FAB**: 우측 하단 화면 위에 부유하는 프라이머리 컬러의 `[ + ]` 방 만들기 버튼 고정 (스크롤 무관).

### 3-2. 컴포넌트 시각적 퀄리티 (Senior Polish)
- **Live Badge (Micro-interaction)**: 활성화된 채널 썸네일이나 카드 외곽에 맥박처럼 뛰는 붉은색/Primary 색상의 `animate-pulse` 테두리나 뱃지 사용.
- **Glassmorphism 맵 배경**: 방 썸네일 이미지 미등록 시, 해당 방 코스의 지도를 배경으로 깔고 CSS `backdrop-blur-md` 유리를 덮어 하이엔드 무드(Premium Mood) 연출.

## 🎫 4. 채널 상세 페이지 (Channel Detail & Room) 기획

목록에서 특정 채널 카드를 클릭했을 때 진입하는 공간으로, 흔한 게시판 형태가 아닌 실시간 액티비티 전후의 **'대기실(Lobby)' 및 'HUD(Head-up Display)'** 개념을 지향합니다.

### 4-1. 방 상태(Status) 기반 UI 변이 (State Machine)
상세 페이지는 채널의 시간적/이벤트 흐름에 따라 전체 레이아웃이 유동적으로 변화해야 합니다.

1. **모집 중 단계 (RECRUITING)**: 
   - 참가자들이 모이는 단계. 참가 버튼 활성화, 호스트와 참가자 간의 소셜 인터랙션(채팅/준비 상태)이 메인 화면.
   - "출발까지 남은 시간" 형태의 카운트다운 타이머 노출.
2. **운동 진행 모드 (LIVE / IN_PROGRESS)**: 
   - 방장(Host)이 '운동 시작'을 누르면 상세 페이지 전체가 **실시간 지도 관제 및 HUD** 모드로 180도 전환.
   - 참가자들의 실시간 GPS 위치 공유와 달성률이 대시보드 형태로 화면을 덮음. (대기실 UI는 축소됨)
3. **종료 단계 (COMPLETED)**: 
   - 운동이 끝난 후 '결과 요약 및 랭보드(운동 궤적, 총 시간, 참여자)' 화면 표시.
   - 1회성 만남이 지속될 수 있도록 서로 팔로우를 맺거나 좋아요(네트워크형 스파크)를 보내는 소셜 액션 유도 후 방 폭파.

### 4-2. 필수 노출 정보 (Lobby UI)

1. **와이드 히어로 영역 (Map / Image Hero)**: 
   - 최상단 배경(`h-[300px]` 이상)에 해당 코스의 궤적이 그려진 지도 스냅샷 또는 커버 이미지 배치.
   - 사진 위에 하단 그라데이션을 넓게 주고, 그 위에 타이틀과 호스트, 코스명 오버레이.
2. **참여자 명단 (Participants Grid)**: 
   - 호스트(왕관 뱃지)를 맨 앞에 두고, 현재 대기실에 입장한 참가자들의 프로필을 가로 스와이프 그리드로 노출.
   - **준비완료 (Ready)** 된 유저의 프로필 테두리에 Primary 색상의 활성 링(Ring)을 띄워 게임 대기방과 유사한 몰입감 제공.
3. **코스 브리핑 및 정보 (Briefing)**: 
   - 타겟 페이스, 예상 소요 시간, 고도(Elevation) 등 객관적 수치 정보와 함께 호스트가 직접 남긴 한 줄 평(예: "워터파크 앞 집결, 천천히 달리는 그룹입니다.") 노출.

### 4-3. 핵심 액션 바 (Sticky Bottom CTA)
- 모바일 기준 화면 하단(`bottom-0`)에 안전 영역까지 가득 채운 고정 액션 바를 두어 명확한 CTA 부여.
- **방장(Host)**: [방 설정 변경], 인원이 준비된 후 활성화되는 **[운동 시작하기]**
- **게스트(Guest)**: 미참여 시 **[참여하기]**, 참여 후엔 **[준비 완료(Ready)]** / [나가기]
- 하단 핵심 버튼 터치 시 무조건 **햅틱 진동 피드백(터치 감각)**을 주어 실제 디바이스를 컨트롤하는 느낌 강화.
