# 🏃‍♂️ 채널 (그룹 운동) UI/UX 기획 및 정책서

**설명**: 실시간 액티비티 기반의 유저 간 그룹 운동 매칭(번개/Flash Mob 방식)을 지원하는 소셜 허브 영역에 대한 디자인 및 비즈니스 정책 문서입니다.

---

## 🎯 1. 핵심 정책 규정 (UX Core)

### 1-1. 휘발성 '방폭형' (1회성 오픈룸 기조)
- 채널은 지속적으로 관리해야 하는 '동호회/크루' 개념을 탈피합니다.
- **"오늘 당장 이 페이스로 같이 뛸 사람"**을 구하는 목적에 부합하는 1회성 룸(Room) 구조를 취합니다.
- 운동 일정 시간이 종료되거나 호스트가 채널을 종료하면 폭파(실시간 지도 추적 종료)되는 가벼운 구조입니다.

### 1-2. 트렌딩 우선 편성과 탐색 (Discovery First)
- 사용자가 하단 GNB를 통해 채널 탭 진입 시, **새롭게 가입할 수 있는 트렌딩 채널 리스트(탐색)**가 디폴트로 활성화됩니다.
- "내가 속한 채널"은 버튼이나 탭 분리로 서브 진입구를 마련하여, 앱 내비게이션의 초기 매력을 즉각적인 '매칭 탐색'에 베팅합니다.

---

## 📊 2. 데이터 노출 우선순위 (UI 레이아웃 기준)

디자이너/퍼블리셔가 채널 목록 카드 컴포넌트를 설계할 때 고려해야 할 요소별 시각적 우선순위입니다.

1. **지역 (Location)**: 
   - 실시간 오프라인 운동 성격장 가장 중요한 기준. 카드 상단 혹은 가장 강력한 아이콘(`MapPin`)과 함께 노출 (예: 한강공원, 남산 둘레길).
2. **현재 접속(활성) 인원 (Live Members)**: 
   - 혼자 뛰는 것이 아님을 알리는 강력한 소셜 자극 장치. (예: 🔴 12/20명)
3. **타겟 정보 (Target Pace / Distance)**: 
   - 나와 운동 레벨(속행)이 맞는지 검토하기 위한 필수 수치 (예: 5'30", 10km)
4. **상태 / 태그 (Status Tag)**: 
   - [오픈/잠금], [러닝/사이클] 구분을 컴팩트한 태그 형태로 표시.

---

## 🎨 3. 레이아웃 및 렌더링 가이드라인 (Design Standard)

### 3-1. 레이아웃 구조 (Mobile / Desktop)
- **상위 컨트롤러**: 상단에 [내 채널] / [탐색(트렌딩)] 탭 스위치를 배치하고, 하단에 즉시 필터 가능한 캐러셀 필터 칩(지역, 거리, 페이스) 배치.
- **메인 리스트**: 카드가 1열(모바일) 또는 2-3열(데스크탑 Grid)로 떨어지는 리스트 뷰 구조. 모바일의 경우 `DESIGN.md` 규칙에 따른 `px-5` 컨테이너.
- **방 생성 FAB**: 우측 하단 화면 위에 부유하는 프라이머리 컬러의 `[ + ]` 방 만들기 버튼 고정 (스크롤 무관).

### 3-2. 컴포넌트 시각적 퀄리티 (Senior Polish)
- **Live Badge (Micro-interaction)**: 활성화된 채널 썸네일이나 카드 외곽에 맥박처럼 뛰는 붉은색/Primary 색상의 `animate-pulse` 테두리나 뱃지 사용.
- **Glassmorphism 맵 배경**: 방 썸네일 이미지 미등록 시, 해당 방 코스의 지도를 배경으로 깔고 CSS `backdrop-blur-md` 유리를 덮어 하이엔드 무드(Premium Mood) 연출.
